A — API (Application Programming Interface)

Planta perenne de tallo leñoso que se ramifica a cierta altura del suelo.

B — Backend

El backend es la parte de un software o aplicación que se ejecuta en el servidor y no es visible para el usuario. Incluye la lógica del sistema, el manejo de bases de datos, autenticación de usuarios, procesamiento de datos, entre otros. Es responsable de que todo lo que el usuario ve en el frontend funcione correctamente.

C — Compilador

Un compilador es un programa que traduce el código fuente escrito en un lenguaje de programación de alto nivel (como C++ o Java) a un lenguaje de bajo nivel (como lenguaje máquina o bytecode) que pueda ser ejecutado por una computadora. A diferencia de los intérpretes, los compiladores procesan todo el programa de una vez.

D — Depuración (Debugging)

Es el proceso de identificar, analizar y corregir errores o fallos (bugs) en el código de un programa. La depuración es fundamental para garantizar que una aplicación funcione correctamente. Herramientas como depuradores (debuggers) y mensajes de error ayudan a los desarrolladores a encontrar rápidamente dónde está el problema.

E — Encapsulamiento

Uno de los principios fundamentales de la programación orientada a objetos. Consiste en ocultar los detalles internos del funcionamiento de un objeto y exponer solo lo necesario a través de interfaces públicas. Esto ayuda a proteger los datos y a reducir la dependencia entre distintas partes del programa.

F — Framework

Un framework es una estructura o conjunto de herramientas y bibliotecas que proporciona una base sobre la cual desarrollar software de manera más eficiente. Define una arquitectura predeterminada y ofrece funcionalidades comunes para que los desarrolladores no tengan que escribirlas desde cero. Ejemplos: Django (Python), Laravel (PHP), Angular (JavaScript).

G - Git

Git es un sistema de control de versiones distribuido que permite a varios desarrolladores trabajar simultáneamente en un proyecto de software, llevando un registro detallado de los cambios realizados en el código a lo largo del tiempo. Es esencial para la colaboración en proyectos y la gestión de versiones del software.

H — HTML

HTML es el lenguaje estándar para crear páginas web. Utiliza etiquetas para estructurar el contenido de una página, como encabezados, párrafos, imágenes y enlaces. Aunque no es un lenguaje de programación, es la base de cualquier sitio web y trabaja junto a CSS y JavaScript.

I — IDE

Un IDE es una aplicación que proporciona un entorno completo para escribir, compilar, depurar y probar código. Generalmente incluye un editor de texto, un compilador o intérprete, y herramientas de depuración.

J — JSON

JSON es un formato ligero de intercambio de datos, muy fácil de leer y escribir tanto para humanos como para máquinas. Es ampliamente utilizado para enviar datos entre un servidor y una aplicación web, y es el formato preferido por muchas APIs por su simplicidad y compatibilidad con JavaScript.

K — Kubernetes

Kubernetes es una plataforma de código abierto para la automatización del despliegue, escalado y gestión de aplicaciones en contenedores. Permite distribuir cargas de trabajo en múltiples máquinas y manejar aplicaciones complejas de forma resiliente, confiable y escalable.

L — Librería

Una librería es un conjunto de funciones, clases o métodos predefinidos que se pueden reutilizar en distintos programas. A diferencia de un framework, una librería es utilizada cuando el desarrollador lo decide, sin imponer una estructura de trabajo.

M — Middleware

Software que se sitúa entre el sistema operativo y las aplicaciones para facilitar la comunicación y gestión de datos. Puede incluir funcionalidades como autenticación, manejo de sesiones, enrutamiento de solicitudes, etc. Es muy común en aplicaciones web modernas.
