Glosario

A B C D E F G H I J K L M N Ñ O P Q R S T U V W X Y Z

N — Node.js

Node.js es un entorno de ejecución para JavaScript que permite usar este lenguaje del lado del servidor. Es conocido por su eficiencia y capacidad para manejar aplicaciones en tiempo real, como chats, servidores web y APIs. Está basado en el motor V8 de Google Chrome.

Ñ — Ñu (GNU)

CGNU es un proyecto de software libre iniciado por Richard Stallman que busca crear un sistema operativo completamente libre compatible con Unix. Es la base del sistema GNU/Linux y ha promovido licencias como la GPL para garantizar la libertad de uso, modificación y distribución del software.

O — Open Source

El software de código abierto es aquel cuyo código fuente está disponible para ser estudiado, modificado y distribuido por cualquiera. Fomenta la colaboración y transparencia en el desarrollo de software.

P — Python

Python es un lenguaje de programación de alto nivel, interpretado, de propósito general y con una sintaxis simple y legible. Es ampliamente utilizado en ciencia de datos, inteligencia artificial, desarrollo web, automatización de tareas, y más.

Q — Query

Una query (consulta) es una instrucción que se envía a una base de datos para solicitar información específica. Las queries se escriben comúnmente en SQL (Structured Query Language) y pueden filtrar, ordenar, agrupar o modificar datos según criterios definidos.

R — React

React es una librería de JavaScript desarrollada por Facebook para construir interfaces de usuario. Permite crear componentes reutilizables que actualizan dinámicamente la vista cuando cambian los datos. Es muy utilizada en el desarrollo de aplicaciones web de una sola página (SPA).

S — Servidor

Un servidor es un sistema (hardware o software) que proporciona servicios o recursos a otros dispositivos (clientes) a través de una red. Puede servir páginas web, almacenar archivos, gestionar correos electrónicos o ejecutar aplicaciones backend.

T — Token

En programación y ciberseguridad, un token es una cadena de caracteres usada para autenticar a un usuario o dispositivo. Puede usarse para sesiones, acceso a APIs o como parte de sistemas de autenticación como OAuth o JWT (JSON Web Tokens).

U — UX (User Experience)

La experiencia de usuario es el conjunto de sensaciones, emociones y percepciones que una persona tiene al interactuar con un sistema digital. Incluye la usabilidad, accesibilidad, diseño, respuesta del sistema, y cómo se siente el usuario al usar una app o sitio web.

V — Variable

Una variable es un contenedor que almacena un valor que puede cambiar durante la ejecución del programa. Tiene un nombre (identificador) y puede contener datos como números, textos, booleanos u objetos, dependiendo del lenguaje.

W — WebSocket

WebSocket es un protocolo que permite comunicación bidireccional, persistente y en tiempo real entre cliente y servidor. A diferencia del HTTP tradicional, WebSocket mantiene abierta una conexión para intercambiar datos sin necesidad de múltiples solicitudes.

X — XML (eXtensible Markup Language)

XML es un lenguaje de marcado que permite estructurar y almacenar datos en forma de texto. Se usa en configuraciones, intercambio de datos entre sistemas y en documentos estructurados. Es legible por humanos y por máquinas, aunque ha sido en gran parte reemplazado por JSON.

Y — YAML (YAML Aint Markup Language)

YAML es un lenguaje de serialización de datos que se utiliza comúnmente para archivos de configuración. Es más fácil de leer que JSON o XML, gracias a su sintaxis basada en la indentación. Se usa mucho en herramientas como Docker, Kubernetes y CI/CD.

Zero-Day

Una vulnerabilidad de día cero es un fallo de seguridad en software que no ha sido descubierto o corregido por el desarrollador. Puede ser explotado por atacantes antes de que exista una solución, lo que lo convierte en una amenaza crítica.
